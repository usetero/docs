Connect GitHub when you want Tero to work with repositories: open pull requests, create GitHub issues, or sync policies for review.

## Connect GitHub

<Steps>
  <Step title="Run Tero setup">
    Open your terminal and run:

    ```bash
    tero
    ```

    Sign in when the setup flow opens your browser.
  </Step>

  <Step title="Select GitHub">
    Choose GitHub as the integration you want to connect.
  </Step>

  <Step title="Install the Tero GitHub App">
    Install the [Tero GitHub App](https://github.com/apps/tero). Select the repositories Tero can access.
  </Step>

  <Step title="Authorize access">
    Review the requested permissions and authorize Tero.
  </Step>

  <Step title="Confirm the connection">
    Tero confirms the installation and can use the selected repositories for configured workflows.
  </Step>
</Steps>

## What GitHub unlocks

GitHub connects policy control to code review.

<CardGroup cols={1}>
  <Card title="Pull requests" icon="code-pull-request" iconType="duotone" horizontal>
    Tero can open PRs when a remediation belongs in source code or repository-managed configuration.
  </Card>
  <Card title="GitHub issues" icon="circle-dot" iconType="duotone" horizontal>
    Tero can create issues when a service owner needs to review or handle a telemetry finding.
  </Card>
  <Card title="Policy sync" icon="rotate" iconType="duotone" horizontal>
    Tero can sync policies to repositories for versioning, review, and audit history.
  </Card>
</CardGroup>

Use GitHub when the right action needs service-owner review, code change, or repository history. Use provider actions or [Edge](/edge/overview) when the right action should run in the telemetry path.

## Permissions

The Tero GitHub App requests these permissions:

| Permission | Access | Why Tero needs it |
| ---------- | ------ | ----------------- |
| **Contents** | Read and write | Read repository context and create branches or commits for configured workflows. |
| **Pull requests** | Read and write | Open PRs and track PR status. |
| **Issues** | Read and write | Create GitHub issues for service-owner review. |
| **Metadata** | Read | Read repository names and basic repository information. |

Limit Tero to the repositories you want it to use during installation. You can update repository access in GitHub App settings.
