## 1. Connect

<Frame>
  <img src="/images/how-it-works-splunk-1.svg" alt="Connect to Splunk" />
</Frame>

Tero connects to Splunk via REST API using an [authentication token](https://docs.splunk.com/Documentation/Splunk/latest/Security/CreateAuthTokens). It works with both Splunk Cloud and Splunk Enterprise. The default setup uses read-only access and leaves your agents, code, and infrastructure unchanged.

## 2. Build context

<Frame>
  <img src="/images/how-it-works-splunk-2.svg" alt="Understand your data" />
</Frame>

Tero analyzes log telemetry and builds the [Master Catalog](/master-catalog/overview). Raw logs group into [log events](/master-catalog/log-events). Tero tracks what each pattern represents, when it matters, and which service produces it.

## 3. Review issues

<Frame>
  <img src="/images/how-it-works-splunk-3.svg" alt="Review issues" />
</Frame>

Tero shows [issues](/issues/overview) with evidence, recommended policies, and related catalog context.

## 4. Take action

Once Tero identifies a problem, you decide what to do.
