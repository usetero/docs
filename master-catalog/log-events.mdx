A log event is a grouped pattern of matching log records.

Tero uses log events to make high-volume telemetry reviewable. Instead of asking you to inspect raw logs one by one, Tero groups records that share a stable pattern and shows the service, severity, volume, cost, policies, and related issues.

## Log event list

The Log events screen can show:

| Field | Description |
| ----- | ----------- |
| Log event | The grouped log pattern name or description. |
| Service | Service that emits the event. |
| Severity | Observed severity for matching logs. |
| Event rate | How often matching logs arrive. |
| Volume | Total observed log volume over the selected window. |
| Cost | Estimated annualized cost for the event. |
| Updated | Most recent update time. |

Datadog-scoped accounts can inspect 24-hour, 7-day, and 30-day log-volume windows.

## Log event detail

Log event detail pages show the event in context. Current detail tabs include overview, policies, and issues.

Use the detail view to inspect:

- The service that emits the event
- Volume and cost context
- Policies attached to the event
- Issues related to the event

## Relationship to issues

Issues use log events as evidence when a finding concerns a recurring log pattern. A cost issue may point to a high-volume event. A compliance issue may point to an event that carries sensitive fields. A policy recommendation may match a specific log event so the runtime can drop, sample, redact, or transform it.

When you review an issue, open the related log event if you need to confirm the pattern, service, volume, or attached policies.
