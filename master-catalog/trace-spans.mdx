Trace spans are not a primary catalog surface in the current Tero navigation.

Use this page only when your Tero account exposes trace policy context or when an issue, policy, or runtime integration refers to trace data.

## Current scope

The current Tero experience is log-oriented: [Issues](/issues/overview), [Policies](/policies), [Services](/master-catalog/services), [Log events](/master-catalog/log-events), log ingestion, Cost, Compliance, Checks, and Edge runtime state.

If an issue or policy references trace context, inspect the issue evidence and policy spec first. This page should not make Trace spans part of the main review flow.

## Trace policy facts

Trace-related policies should identify:

- Span or operation name
- Producing service
- Attributes involved in the finding
- Current action, such as filtering or redaction
- Runtime path that applies the policy
